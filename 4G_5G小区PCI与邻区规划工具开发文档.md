# 网优百宝箱 v1.2.1 - 开发文档

> 项目原名"4G/5G小区PCI与邻区规划工具"，自 v1.2.1 起更名为 **网优百宝箱**。

## 1\. 项目概述

### 1\.1 项目目标

开发一款轻量化、可离线部署的网优百宝箱（4G/5G小区规划工具），支持工参导入、GIS可视化、**三级PCI智能规划**、**距离70%\+扇区交叠30%加权邻区规划**、多制式邻区生成、冲突校验、报表与MML脚本导出，适配陆地及近海超远覆盖场景，替代传统人工规划、零散脚本工具。

### 1\.2 核心功能清单

- 工参Excel/CSV导入、数据清洗、异常校验

- Leaflet地图可视化：基站点位、扇区覆盖、冲突标红、邻区连线

- PCI分级规划：全网粗规划→局部微调→二次冲突校验（融合开源算法最优逻辑）

- 智能邻区规划：同频/异频/4G↔5G异系统邻区、7:3加权打分、冗余过滤、数量限制、双向邻区补齐

- 全网冲突检测：同PCI、Mod3、Mod30冲突核查

- 规划结果导出：新工参表、冲突报表、邻区清单、运营商MML脚本

- 近海场景适配：可自定义超远覆盖距离阈值、独立PCI资源池

### 1\.3 技术栈选型

#### 前端（可视化层）

- 基础框架：原生Vue3 / 原生JS（轻量无冗余）

- 地图引擎：Leaflet（轻量开源、加载快、适配离线）

- 地理计算：Turf\.js（扇区生成、多边形交叠、空间计算）

- UI组件：Element Plus（表单、弹窗、文件上传、数据表格）

#### 后端（算法服务层）

- 服务框架：FastAPI（轻量接口、高性能、自动接口文档）

- 核心语言：Python3\.9\+

- 地理算法：Vincenty椭球距离计算（替代Haversine，适配超远场景）

- 数据处理：Pandas、Numpy（工参解析、批量计算）

- 打包部署：PyInstaller（支持离线EXE单机部署）

#### 算法溯源（开源项目剥离整合）

- PCI核心逻辑：剥离 **lte\-pci\-planner** 贪心分配 \+ **rf\-planning** 分级规划、冲突校验逻辑

- 邻区核心逻辑：剥离 **nr\-neighbor\-planner** 多制式邻区、双向邻区、冗余过滤逻辑，自研7:3加权打分模型

- 地理底层：全网通用Vincenty椭球算法，适配近海超远距离计算

## 2\. 系统整体架构

系统采用**前后端分离轻量架构**，无数据库依赖（本地文件存储），支持离线运行，分为三层：

### 2\.1 前端展示层

负责用户交互、地图可视化、参数配置、文件上传、结果预览、数据导出，不处理复杂算法，仅做页面渲染与接口请求。

### 2\.2 后端算法服务层

核心业务层，包含地理计算、PCI规划引擎、邻区规划引擎、数据解析、冲突校验、脚本导出六大核心模块，接收前端参数执行批量计算并返回结果。

### 2\.3 数据持久层

本地文件存储，支持Excel/CSV读写、临时缓存规划结果，无需安装数据库，轻量化免部署。

## 3\. 核心模块详细设计

### 3\.1 基础地理计算模块（底层公共模块）

全局所有规划算法依赖的基础工具，统一封装，保证全网计算精度一致。

#### 核心能力

- Vincenty椭球距离计算：精准计算两点经纬度距离（支持0\-100km超远小区，适配近海场景）

- 扇区多边形生成：经纬度\+方位角\+覆盖半径，生成扇形覆盖区域

- 扇区交叠面积计算：Turf\.js空间几何运算，获取两小区覆盖重叠面积

- 方位角夹角校验：快速过滤无交叠、无邻区价值的小区，减少计算量

### 3\.2 PCI分级规划模块（核心业务）

整合开源项目最优逻辑，自研三级规划流程，兼顾全网最优与局部改动最小化。

#### 3\.2\.1 规划流程（标准三级流程）

1. **全网粗规划**：地理分区\+独立PCI资源池，贪心算法分配，最大化同PCI复用距离，规避基础冲突

2. **局部微调规划**：支持框选新增小区，仅重算周边辐射范围小区PCI，保留存量合规小区不变，降低网规改动成本

3. **全局二次校验**：全量遍历冲突，自动修复轻度冲突，输出冲突清单

#### 3\.2\.2 冲突校验规则

- 4G LTE：同PCI冲突、Mod3时序混淆冲突

- 5G NR：同PCI冲突、Mod3时序冲突、Mod30 DMRS信道冲突

#### 3\.2\.3 PCI资源池规则

- 4G PCI：0\~503 全局资源池，分区复用

- 5G PCI：0\~1007 全局资源池，严格规避Mod30冲突

- 支持自定义PCI白名单、黑名单、固定PCI小区锁定

### 3\.3 加权智能邻区规划模块（核心自研）

基于开源邻区框架，替换核心打分逻辑，实现**距离70% \+ 扇区交叠30%**权重模型，支持全制式邻区覆盖。

#### 3\.3\.1 核心打分公式

归一化处理后统一0\-1分值：
总得分 = \(1 \- 归一化距离分\) × 0\.7 \+ 归一化交叠面积分 × 0\.3
**规则**：距离越近、交叠面积越大，邻区优先级越高

#### 3\.3\.2 邻区全功能能力

- 多制式支持：同频邻区、异频邻区、4G↔5G NSA/SA异系统锚点邻区

- 阈值过滤：可自定义最大邻区距离、最小交叠面积，过滤无效邻区

- 数量限制：单小区可配置最大邻区数量，避免冗余超标

- 双向邻区自动补齐：A配置B为邻区，自动生成B反向邻区关系

- 外部邻区管理：跨TAC、跨站点邻区分组统计、单独导出

- 冗余过滤：低分邻区、超远邻区、无交叠邻区自动剔除

### 3\.4 数据解析与导出模块

#### 导入能力

支持标准网优工参Excel/CSV，字段包含：ECGI、小区名称、制式、频点、经度、纬度、方位角、覆盖半径、TAC、PCI、站点类型

#### 导出能力

- 规划后完整工参表（含新PCI、邻区清单）

- 全网PCI冲突核查报表

- 邻区冗余/漏配检测报表

- 华为/中兴通用MML开通脚本（适配国内运营商）

## 4\. 核心算法伪代码与实现规范

### 4\.1 邻区加权打分算法

```Plain Text
def calc_neighbor_score(distance: float, overlap_area: float, max_dist_threshold: float, max_overlap: float) -> float:
    # 归一化计算 0~1
    dist_norm = 1 - min(distance / max_dist_threshold, 1.0)
    overlap_norm = min(overlap_area / max_overlap, 1.0)
    # 7:3权重配比
    total_score = dist_norm * 0.7 + overlap_norm * 0.3
    return round(total_score, 4)

```

### 4\.2 PCI冲突检测算法

```Plain Text
# 4G LTE冲突检测
def check_lte_conflict(pci_src: int, pci_dst: int) -> (bool, str):
    if pci_src == pci_dst:
        return True, "同PCI冲突"
    if pci_src % 3 == pci_dst % 3:
        return True, "Mod3时序冲突"
    return False, "无冲突"

# 5G NR冲突检测
def check_nr_conflict(pci_src: int, pci_dst: int) -> (bool, str):
    if pci_src == pci_dst:
        return True, "同PCI冲突"
    if pci_src % 3 == pci_dst % 3:
        return True, "Mod3时序冲突"
    if pci_src % 30 == pci_dst % 30:
        return True, "Mod30 DMRS冲突"
    return False, "无冲突"

```
